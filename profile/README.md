# Astropods

Astropods is a production-grade control plane for AI agents. It grew out of building real agents in production and is now the foundation for making agents discoverable, governable, and deployable as first-class operational assets inside your organization.

## What's in this org

| Repo | Description |
|---|---|
| [`agents`](https://github.com/astropods/agents) | A catalog of ready to use agents|
| [`adapters`](https://github.com/astropods/adapters) | Agent adapters for connecting to internal systems and external APIs |
| [`messaging`](https://github.com/astropods/messaging) | Production-ready Go messaging service for agent-to-platform communication |
| [`playground`](https://github.com/astropods/playground) | Standalone web chat interface for interacting with agents |
| [`ast-cli`](https://github.com/astropods/ast-cli) | The future home for the CLI |

## What Astropods do

An Astropods packaged agent is a versioned harness that includes its execution graph, tool contracts, data substrate, guardrail policies, observability configuration, and deployment metadata. The format is open, model-agnostic, and portable across runtimes.

The platform gives you:

- **Reusable primitives** for data ingestion and connecting agents to internal systems and external APIs, so teams stop rebuilding the same plumbing
- **Consistent observability and guardrails** so you know what's running, where, and under what constraints
- **A structured development workflow** from local scaffolding to production deployment, reducing the mechanical overhead that slows down experimentation and iteration
